Документация по Graph BFS

Класс Graph

Представляет ориентированный граф с использованием матрицы смежности и предоставляет функциональность обхода в ширину (BFS).

Конструктор

Graph()

Создаёт новый экземпляр класса Graph.

Деструктор

~Graph()

Очищает ресурсы, используемые объектом Graph.

Метод: loadFromFile

bool loadFromFile(const string& fileName)

Загружает граф из файла, содержащего матрицу смежности.

Параметры: fileName – имя файла с данными графа.

Возвращает: true — если загрузка прошла успешно, false — в случае ошибки (файл не найден или данные некорректны).

Метод: breadthFirstSearch

vector<int> breadthFirstSearch(int startVertex)

Выполняет обход графа в ширину (BFS), начиная с указанной вершины.

Параметры: startVertex – стартовая вершина (нумерация с 0).

Возвращает: Вектор достижимых вершин в порядке BFS. Вершины с равным приоритетом сортируются по возрастанию.

Исключение: invalid_argument, если startVertex некорректен (отрицательное значение или больше числа вершин).

Метод: clear

void clear()

Очищает внутренние структуры графа и сбрасывает количество вершин в 0.

Метод: isValidVertex

bool isValidVertex(int vertex) const

Проверяет, существует ли вершина в текущем графе.

Параметры: vertex – вершина для проверки.

Возвращает: true — если вершина существует, false — в противном случае.

Главная функция

int main()

Точка входа в программу, обрабатывающая взаимодействие с пользователем и координирующая работу с графом.

Последовательность выполнения:

  1. Запрашивает у пользователя имя входного файла
  2. Запрашивает стартовую вершину (нумерация с 0)
  3. Загружает граф из указанного файла
  4. Выполняет обход в ширину (BFS)
  5. Выводит достижимые вершины в порядке BFS

Требования к формату файла:

  • Первая строка: количество вершин (n)
  • Следующие n строк: матрица смежности n×n, где:
    • 0 — отсутствие ребра
    • Ненулевые значения — наличие ребра (вес не учитывается)

Особенности реализации

  • Граф представлен через матрицу смежности
  • BFS реализован с использованием очереди и массива посещённых вершин
  • Вершины с одинаковым приоритетом сортируются по возрастанию
  • Реализована обработка ошибок при неверных вершинах и файлах
